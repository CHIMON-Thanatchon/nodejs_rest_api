# ลำดับการอ่านเอกสาร (อย่าข้าม)

อ่านไฟล์ตาม **เลขลำดับ** เพื่อให้ flow การทำงานของเซิร์ฟเวอร์ (จาก bootstrap → DB → session → auth → โดเมน) ต่อเนื่องกับสมอง

| ลำดับ | ไฟล์ | เรื่อง |
|------|------|--------|
| **1** | [01-nest-fundamentals-structure-tools.md](./01-nest-fundamentals-structure-tools.md) | แนวคิด Nest, โครงสร้างโปรเจกต์นี้, เครื่องมือหลัก |
| **2** | [02-main-bootstrap-and-global-setup.md](./02-main-bootstrap-and-global-setup.md) | `main.ts`: สร้างแอป, session, cookie, pipe, CORS |
| **3** | [03-app-module-config-typeorm.md](./03-app-module-config-typeorm.md) | `app.module.ts`: Config, TypeORM, โมดูลย่อย, Global Guard |
| **4** | [04-environment-validation.md](./04-environment-validation.md) | `.env`, Joi validation, `env.util` |
| **5** | [05-entities-and-database.md](./05-entities-and-database.md) | Entity, ตาราง PostgreSQL, ความสัมพันธ์ |
| **6** | [06-express-session-and-connect-pg-simple.md](./06-express-session-and-connect-pg-simple.md) | Session + cookie + เก็บ session ใน DB |
| **7** | [07-auth-module-jwt-and-session.md](./07-auth-module-jwt-and-session.md) | สมัคร/ล็อกอิน, JWT, Passport Strategy |
| **8** | [08-guards-and-decorators.md](./08-guards-and-decorators.md) | `JwtAuthGuard`, `RolesGuard`, `@Public()`, `@Roles()` |
| **9** | [09-users-module.md](./09-users-module.md) | UsersService, การค้นหา/สร้างผู้ใช้ |
| **10** | [10-books-module.md](./10-books-module.md) | หนังสือ, seed, สิทธิ์ admin |
| **11** | [11-cart-module.md](./11-cart-module.md) | ตะกร้าใน session |
| **12** | [12-orders-module.md](./12-orders-module.md) | เช็คเอาต์, ทรานแซกชัน, ประวัติคำสั่งซื้อ |

หลังอ่าน **ข้อ 1** แล้วค่อยไป **ข้อ 2** ต่อเรื่อย ๆ จะเข้าใจ path การรันคำขอ (request) จาก client จนถึง DB ได้ชัดที่สุด
